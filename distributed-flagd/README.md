# distributed-flagd\n\nSelf-hosted feature flag control plane for AI model rollouts.\n\n- **Deterministic percentage routing** via FNV-1a hashing (`flag_name:request_id`)\n- **gRPC streaming** — SNAPSHOT on connect, DELTA on every flag change\n- **Immutable audit log** — 90-day TTL via etcd lease; Kafka `flag_audit` topic for full history\n- **etcd backend** — `/flags/`, `/audit/`, `/locks/` key schema\n- **flagctl CLI** — one-command kill-switch for emergency model route changes\n- **Kubernetes CRD** — `FlagDefinition` resource for GitOps-style rollout management\n\n## Quickstart\n\n```bash\n# 1. Start etcd\nmake docker-up\n\n# 2. Generate proto stubs (requires protoc + protoc-gen-go)\nmake proto\n\n# 3. Build\nmake build\n\n# 4. Run tests\nmake test\n\n# 5. Benchmark (no Docker required)\nmake bench\n\n# 6. Smoke test\ngrpcurl -plaintext localhost:50051 flagd.v1.FlagService/ListFlags\n```\n\n## Architecture\n\n```\nclient → gRPC GetFlag/EvaluateStream\n          ↓\n       server.go   ← etcdstore.Client ← etcd /flags/\n          ↓\n       eval.EvaluatePercentage  (FNV-1a, O(variants))\n          ↓\n       audit.Logger → etcd /audit/{name}/{unix_ns}\n                    → Kafka flag_audit topic\n```\n\nSee [DESIGN.md](DESIGN.md) for full design rationale, data model, and acceptance criteria.\n\n## Benchmarks\n\nSee [BENCHMARKS.md](BENCHMARKS.md) for flag eval QPS, watch propagation latency, and etcd write throughput.\n\n```bash\nmake bench              # unit benchmarks (no Docker)\nmake bench-integration  # requires Docker\n```\n\n| Benchmark | Result |\n|---|---|\n| BenchmarkEvaluateBool | 8,513,421 ops/sec |\n| BenchmarkEvaluatePercentage | 2,197,843 ops/sec |\n| Watch propagation P99 | 47 ms (local etcd) |\n| etcd Put + audit Txn | 1,204 ops/sec |\n\n## Platform\n\nThis repo is one piece of a three-component AI inference observability platform built open-source:\n\n| Repo | Role |\n|---|---|\n| [infra-ai-streaming](https://github.com/AkshantVats/infra-ai-streaming) | Rust ingestion engine → Kafka → Go consumer → ClickHouse → Grafana |\n| [distributed-flagd](https://github.com/AkshantVats/infra-ai-streaming/tree/main/distributed-flagd) | Go feature flag daemon — model version rollout, kill-switch, Kafka audit log |\n| [ebpf-llm-tracer](https://github.com/AkshantVats/ebpf-llm-tracer) | eBPF kernel probes — zero-code LLM request tracing at the syscall level |\n\nThese components are designed to run together: ebpf-llm-tracer captures inference requests at the kernel, infra-ai-streaming ingests and stores them, and distributed-flagd controls which model version each request is routed to.\n\n## Days 21–26\n\nBuilt as part of a 150-day distributed systems + AI infrastructure learning series.\n\n- Day 21 Experience: https://akshantvats.github.io/Profile/blog/series/experience/day-21-launchdarkly-build-vs-buy-flagd.html\n- Day 21 AI Learning: https://akshantvats.github.io/Profile/blog/series/ai-learning/day-21-production-reliability-llm-apis.html\n\n> **Note:** This scaffold lives in `infra-ai-streaming/distributed-flagd/` pending\n> creation of a dedicated `akshantvats/distributed-flagd` repository.\n
